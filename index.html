What this is

This is a fork off of EVE — the EVE-side baseline. We're using it to clear up the requirements and stand up our initial demo. That's all it is.

A few things to be clear on:

  • All Epiq data stays inside the Epiq environment. The demo data lives in Postgres, inside Epiq infrastructure — none of Epiq's data is ever handled outside it.
  • No proprietary Epiq information belongs in this repo. The client-confidential side is set up and governed by Tobias, walled off; we contribute to that through him.
  • Once the demo is approved, it becomes a repository build on Epiq infrastructure. The EVE fork can be copied back into the Epiq environment and we continue there.
  • We're not here to own the capital asset — that's Epiq's.

What you're joining

The revenue-leakage model lives in one public GitHub repository. Everyone works off the same source of truth. You bring your own surface — the AI tool you like best — point it at the repo, and your work flows back in.

Start with the foundation Tobias set: Document A — Revenue Leakage Corrections (the anchor doc), then the consolidated model canon.

Step by step

1

Get a free GitHub account

If you don't have one yet, sign up. Takes two minutes.

Create GitHub account
2

Tell the team your GitHub username

Send your username to the WhatsApp group so you can be added as a collaborator. Once you're added, you can push your own work back into the repo.

3

Get the repository onto your machine

Open a terminal and run:

git clone https://github.com/EVEglyphDesign/epiq-revenue-leakage.git
cd epiq-revenue-leakage

No terminal? You can also open the repo in the browser and use GitHub's built-in editor — press . (the period key) on the repo page.

4

Pick your surface

Point whichever tool you prefer at the repo folder. Any of these work — pick one:

Cursor

Open the folder in Cursor and ask it about MODEL-CANON.md.

VS Code + Copilot

Open the folder, let Copilot read the repo as context.

Claude / ChatGPT

Paste in MODEL-CANON.md and work from there.

GitHub web editor

Edit right in the browser — no install needed.

5

Spin the model, then push it back

Make your changes, then send them back so the whole team — and this page — stays current:

git add .
git commit -m "your note here"
git push

Using the browser editor? Just hit Commit changes. That's it.

M

Matt — bring the billing operation in

We want to twin the whole operation Jackie runs, as fast as possible. When you connect, add everything you have on the draft billing (Fury) app — especially where the table structures live and what they look like. Drop it into a billing-twin/ folder in the repo.

Include, as best you can:

  • Fury app data model — table names, keys, relationships
  • Any schema exports, DDL, or screenshots of the table structures
  • The SAP data the billing operation depends on

The goal: take all the SAP data requirements and pipeline them through DataSphere at once, so the billing operation twins alongside the leakage model instead of in a second pass.

6

Grow your own digital twin

Once you're pushing work, you're building your own surface on top of the shared model — the same way the rest of us do. Add your own folder, keep the model canon intact, and keep pace as we go.
